Something Useful

Jan 13th, 2011

Well.

I worked on that scraper for a bit tonight, and finally have something that I will actually use now.  You can check it out HERE.  A few notes about it:

  • I did not go and learn the correct way to program in Ruby, as I claimed I would before finishing this. Although I did learn some basic Ruby syntax in the work I’ve done on this so far, I mostly grew impatient and decided to hack out a solution tonight.
  • No really, this is a hack. I decided I wanted sorting so I quickly found a javascript table sorter and based my output around that code (which is most noticeable with the * before January postings [yay alphabetic sorting!]).
    • The date was particularly problematic, considering that the page I was parsing gave dates in a %a %b %d format (i.e. ‘Thu Jan 10’). This meant I could either write my own code to figure out the year (which would still be a hack, and vary depending on the current date) or just hack it like I did (albeit temporarily)
  • Relating to that last point, I was considering storing this data in a database. This would make sorting a simple matter of changing the query, but that kind of stuff is easy and wouldn’t really help me or the page in any significant way; that’s probably not going to happen.
  • The script as it stands is just writing an HTML file from its data. While I could have had it do this in-place, I’m actually storing the jobs individually as objects in an array, and then later writing them out. This is obviously wasteful but again, at this scale who cares (as long as I’m aware of it, right?).
  • I set up the script to run every half-hour during normal business hours. I thought that was cool.
